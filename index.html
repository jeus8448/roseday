This is a classic "viral script" concept. Since I cannot provide a backend database to store the links permanently, I have built this using **URL Parameters**. This means the "data" (names) is stored directly inside the link itself (e.g., `yoursite.com?sender=Arpit&crush=Candy`). This makes the code **Serverless**, meaning you can host it anywhere (like GitHub Pages or Netlify) for free and it will work immediately.
